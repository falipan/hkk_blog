docker和phpstorm搭建php开发环境

首先docker-compose启动php5.6、php7.4、mysql5.6三个容器，并建立连接

version: '3'
services:
    php5.6:
        image: php:5.6-apache 
        ports: 
            - 8000:80 
        volumes: 
            [localfile:/var/www/html]
        links: 
            - mysql
        stdin_open: true
        tty: true
    php7.4:          
        image: php:7.4-apache  
        ports: 
            - 80:80 
        volumes: 
            [localfile:/var/www/html]
        links: 
            - mysql 
        stdin_open: true
        tty: true

    mysql:
        image: mysql:5.6
        environment: 
            MYSQL_ROOT_PASSWORD: "313807"
        ports: 
            - 3306:3306

服务起来后

进入php5.6与php7.4容器更新并安装vim

apt update

apt install vim

将php-ini-production复制到php.ini

cp /usr/local/etc/php/php.ini-production /usr/local/etc/php/php.ini

docker中安装php扩展使用命令

docker-php-ext-install

php启动扩展

docker-php-ext-enable

安装mysql与mysqli扩展

docker-php-ext-install mysql

docker-php-ext-install mysqli

安装xdebug扩展

pecl install xdebug

pecl install xdebug-2.5.0(5.6需安装此版本)

启动php扩展

docker-php-ext-enable mysql

docker-php-ext-enable mysqli

docker-php-ext-enable xdebug

配置php.ini

echo -e 'xdebug.remote_enable=On
xdebug.remote_host= [remote_ip]
xdebug.remote_port=9000
xdebug.auto_trace=On
xdebug.collect_vars=On
xdebug.collect_return=On
xdebug.collect_params=On
xdebug.idekey=phpstorm
xdebug.remote_log="/tmp/xdebug.log"' >> /usr/local/etc/php/php.ini

将remote_ip替换为远程ip

php5.6由于mysql，mysqli的扩展为.so文件，而php.ini中为.dll所以需要更改php.ini

extension=mysql.so
extension=mysqli.so

php7.4只需删除extension前;即可

以上步骤完全可以通过docker-compose与Dockerfile实现，出于懒
